❓ undefined vs null ⭕

JavaScript-тегі ең шатастыратын екі түр

❓
undefined
"Мен бар, бірақ менің мәнім жоқ"
Айнымалы жарияланған, бірақ мән берілмеген
let name; console.log(name); // undefined let obj = {}; console.log(obj.age); // undefined function sayHello() { // return жоқ } console.log(sayHello()); // undefined
Қашан пайда болады?
• Мән берілмеген айнымалы
• Жоқ объект қасиеті
• return жоқ функция
• Массивтегі жоқ элемент
VS
⭕
null
"Мен қасақана бос етілдім"
Программист қасақана бос мән берді
let data = null; // қасақана бос let user = getCurrentUser(); if (!user) { user = null; // пайдаланушы жоқ } // API жауабы let response = { data: null, error: "Пайдаланушы табылмады" };
Қашан пайдаланамыз?
• Қасақана "бос" мән
• API жауаптарында
• Айнымалыны "тазалау"
• Объектті жою алдында

💻 Практикалық мысалдар

🎯 Тапсырма 1: Тексеру функциясы

Мәннің undefined немесе null екенін тексеретін функция жазыңыз:

function isEmpty(value) { return value === undefined || value === null; } // Немесе жаңа тәсіл: function isEmpty(value) { return value == null; // undefined және null екеуін де тексереді }

🔍 Тапсырма 2: Қауіпсіз қасиетке қол жеткізу

Объект қасиетіне қауіпсіз түрде қол жеткізіңіз:

// Дұрыс емес тәсіл: // console.log(user.profile.name); // Error егер profile undefined болса // Дұрыс тәсіл: console.log(user.profile?.name); // Optional chaining // Немесе: if (user.profile !== undefined && user.profile !== null) { console.log(user.profile.name); }

🎨 Тапсырма 3: Default мәндер

undefined/null болғанда default мән беріңіз:

function greetUser(name) { // Nullish coalescing operator (??) name = name ?? "Қонақ"; return "Сәлем, " + name; } // Немесе: function greetUser(name = "Қонақ") { return "Сәлем, " + name; }
